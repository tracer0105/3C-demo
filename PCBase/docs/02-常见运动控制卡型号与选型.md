# 02 常见运动控制卡型号与选型

## 1. 先说明白：为什么这里不只写“插卡”

老一代运动控制项目里，“运动控制卡”通常特指 `PCI/PCIe` 插卡式板卡。  
但在今天的 PC-based 运动控制项目里，常见方案已经扩展为：

- 传统 PCI/PCIe 脉冲或模拟量运动控制卡
- PCIe EtherCAT 主站卡
- IPC + 软运动平台
- 通过以太网连接 PC 的网络型运动控制卡

所以本章采用“代表性型号 + 工程归类”的方式整理，既方便学习，也更接近真实选型场景。

## 3. 代表型号说明

### 3.1 研华 Advantech

#### `PCI-1245E`

适合理解传统脉冲型 PC-based 运动控制的基本盘：

- 4 轴通用 PCI 运动控制卡
- 脉冲输出最高可到 `5 Mpps`
- 支持 `E-Gear`
- 带硬件急停输入、看门狗、位置锁存
- 使用统一的 `Common Motion API`

适合谁：

- 第一次做 2 到 4 轴控制
- 要维护老设备
- 想从脉冲控制入门

#### `PCIE-1245` / `PCIE-1245E`

这是更贴近今天 IPC 设备的 PCIe 版本：

- `PCIE-1245` 支持点位、直线、圆弧、螺旋插补和轨迹缓存
- `PCIE-1245E` 定位在经济型方案
- 官方资料中已经采用 `MotionNavi API` / 通用运动架构

如果你的工控机已经没有 PCI 只剩 PCIe，这一类产品更现实。

### 3.2 凌华 ADLINK

#### `PCI-8254/8258`

这是典型的高性能集中式运动控制卡：

- `PCI-8254` 为 4 轴，`PCI-8258` 为 8 轴
- 支持模拟量和脉冲混合控制
- 官方强调 `PID + Feed Forward` 的闭环控制
- 配套 `MotionCreatorPro 2`

适合谁：

- 需要更高控制性能的多轴平台
- 对闭环精度、同步性能要求较高

#### `PCIe-8338`

这是 EtherCAT 主站卡路线的代表型号：

- 最多支持 `64` 个同步轴
- 可管理 `10000+` I/O 点
- EtherCAT 周期时间可到 `250 us`
- 带专用急停输入

适合谁：

- 8 轴以上伺服设备
- 需要远程 I/O、驱动器参数化、总线诊断
- 要做更强同步控制的设备

### 3.3 固高 GOOGOLTECH

固高的 `GTS` 系列在国内运动控制领域非常常见，尤其适合作为“传统插卡式运动控制”的学习样本。

以 `GTS-400-PCIe` 系列为例：

- 基于 `PCIe` 总线
- 4 轴控制
- 可分为开环与闭环/混合类型
- 通过 `DSP + FPGA` 做运动规划
- 支持点位、连续轨迹、多轴同步、直线/圆弧/螺旋插补
- 官方提供 `VC、VB、C#、LabVIEW` 等开发库

常见学习型号：

- `GTS-400-PG-PCIe`：开环脉冲型
- `GTS-400-PV-PCIe`：闭环/混合型
- `GTS-400-GT-PCIe`、`GTS-400-VT-PCIe`：增强版系列

### 3.4 正运动 ZMotion

#### `ECI2828-V2`

这是一类很适合解释“现代网络型运动控制卡”的产品：

- 基本 `8` 轴，可扩到 `16` 轴
- 同时支持 `EtherCAT` 与 `Pulse`
- 带 `RS232、Ethernet、EtherCAT、CAN`
- 支持电子凸轮、电子齿轮、同步跟随、虚拟轴
- 官方提供 `zmotion.dll`、`ZDevelop` 和多种语言开发示例

适合谁：

- 不想强依赖 PCI/PCIe 插槽
- 要做中小型 EtherCAT 或混合控制项目
- 希望用 C/C#/Python/Qt 等语言快速接入

### 3.5 Beckhoff / Omron PMAC / Galil

这一组更像“平台级”方案，不只是单张卡：

#### Beckhoff `TwinCAT 3 + EtherCAT`

- 强项是统一的平台化能力
- 可以从简单 `PTP` 扩展到 `CNC`、`Robotics`
- 把 PLC、运动控制、视觉和通信放到同一 PC 软件平台

#### Omron `Power UMAC` / `Power Brick AC ARM`

- 更偏高端、复杂设备控制
- 开放架构，模块化强
- 适合复杂多轴与定制化运动控制场景

#### Galil `DMC-40x0`

- 强调高性能多轴控制
- 支持协调运动与 `PVT`
- 配套 `GDK` 和 `gclib`

如果你的项目要走平台化、可扩展、多代机型复用路线，这三类方案很值得学习。

## 4. 怎么根据需求快速选型

### 场景 A：2 到 4 轴，成本敏感，逻辑不复杂

优先考虑：

- 传统脉冲型 `PCI/PCIe` 控制卡
- 例如 `PCI-1245E`、`PCIE-1245E`、`GTS-400-PG-PCIe`

适合：

- 点位平台
- 搬运机构
- 步进或脉冲伺服项目

### 场景 B：4 到 8 轴，要求更好的同步和闭环性能

优先考虑：

- 闭环/混合型运动控制卡
- 例如 `PCI-8254/8258`、`GTS-400-PV-PCIe`

适合：

- 双工位协同
- 高精度平台
- 需要编码器反馈和更复杂轨迹的设备

### 场景 C：8 轴以上，伺服多、I/O 多、诊断要求高

优先考虑：

- EtherCAT 主站卡或软运动平台
- 例如 `PCIe-8338`、`TwinCAT 3 + EtherCAT`

适合：

- 3C 自动化线
- 高速装配设备
- 多模组联动平台

### 场景 D：不想被插槽限制，希望布线和扩展更灵活

优先考虑：

- 网络型运动控制卡
- 例如 `ECI2828-V2`

适合：

- 中小型设备
- 非标准工控机架构
- 希望以以太网连接上位机

## 5. 选型时必须核对的参数

不要只看“几轴”。至少核对下面这些项：

| 维度 | 为什么重要 |
| --- | --- |
| 控制方式 | 脉冲、模拟量、EtherCAT 决定了布线、同步能力和系统复杂度 |
| 实际轴数与可扩展轴数 | 当前项目够不够、下一代机型还能不能复用 |
| 反馈接口 | 编码器、光栅尺、绝对值反馈能不能接 |
| 插补能力 | 是否支持直线、圆弧、螺旋、空间插补 |
| 比较输出/锁存 | 视觉飞拍、打标、点胶、测长常会用到 |
| 同步周期 | 对 EtherCAT 或高性能闭环项目尤其重要 |
| SDK 语言支持 | 你是 C#、C++、LabVIEW 还是 Python 项目 |
| 调试工具 | 有没有官方示波器、诊断工具、对象字典工具 |
| 操作系统支持 | Windows 10/11、驱动签名、32/64 位 |
| 本地支持 | 文档、代理商、FAE、备件是否充足 |

## 6. 工程上更实用的选型原则

### 原则 1：先看“你的驱动器”，再看“你的卡”

运动控制卡不是孤立采购的。必须先确认：

- 驱动器接收脉冲、模拟量还是 EtherCAT
- 编码器是接驱动器还是接控制卡
- 驱动器是否支持 `CiA 402`
- 现场是否需要远程 I/O 和安全模块

### 原则 2：先做最小样机，再批量导入

建议先做一个 POC：

- 单轴或双轴联调
- 回零
- 限位
- 绝对移动
- 报警恢复
- 一段简单插补

如果样机阶段都不稳定，不要直接大批量导入整机项目。

### 原则 3：新项目优先考虑可扩展平台

如果你预计后面会接：

- 视觉
- MES
- 配方
- 轨迹文件
- 工艺追溯
- 远程维护

那就不要只看“这张卡便不便宜”，而要看整个软件栈是否能支撑后续扩展。

## 7. 给初学者的结论

如果只是为了入门学习：

- 想学传统脉冲控制：看 `PCI-1245E`、`PCIE-1245`、`GTS-400` 系列
- 想学高性能闭环：看 `PCI-8254/8258`
- 想学现代总线型方案：看 `PCIe-8338`、`ECI2828-V2`、`TwinCAT 3`

如果是要做真实项目：

- 优先选官方资料齐全、示例完整、售后支持强的方案
- 优先选你团队能稳定维护的 SDK，而不是功能表最华丽的那张卡
