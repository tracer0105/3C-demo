# 01 认识 PCBase 与运动控制

## 1. 什么是 PCBase

在自动化行业里，很多人会把“PCBase”理解成“以 PC 为核心的控制系统”，也就是 `PC-based control` 或 `PC-based motion control`。

它的核心思想是：

- 用工业 PC 或工控机承载上位机界面、逻辑控制、数据处理、视觉、配方和通信
- 用运动控制卡、EtherCAT 主站卡或软运动平台来完成多轴运动控制
- 通过驱动器和电机把软件命令变成真实机械动作

这类架构常见于以下设备：

- 点胶、锁螺丝、焊锡、贴装、搬运等 3C 设备
- AOI、激光、点位平台、龙门模组
- 包装、木工、数控、半导体设备
- 需要多轴同步、视觉联动、数据采集、工艺追溯的机器

## 2. PC-based 运动控制系统由什么组成

一个典型的系统通常由以下部分组成：

| 组件 | 作用 | 初学者关注点 |
| --- | --- | --- |
| 工业 PC / IPC | 跑上位机、数据库、HMI、视觉、通信 | CPU、网口、扩展槽、系统稳定性 |
| 运动控制卡 / 主站卡 / 软运动平台 | 生成轨迹、发脉冲、做同步控制或管理 EtherCAT 总线 | 轴数、总线、周期、SDK、示例 |
| 伺服驱动器 / 步进驱动器 | 把控制命令转成电机电流、电压与运动 | 控制模式、编码器兼容性、报警接口 |
| 电机 | 真正执行转动或直线运动 | 功率、惯量、速度、扭矩 |
| 编码器 / 光栅尺 | 提供位置或速度反馈 | 分辨率、绝对值/增量、接口类型 |
| I/O 模块 | 接急停、限位、气缸、传感器、按钮等 | 输入输出点数、隔离、响应速度 |
| 安全链 | 急停、门禁、STO、安全继电器 | 不要把软件停止当作安全停止 |
| 机械机构 | 丝杆、皮带、直线模组、龙门、夹具 | 刚性、间隙、行程、负载 |

## 3. 常见控制架构

### 3.1 脉冲型控制卡

特点：

- 控制卡输出 `Pulse/Dir` 或 `CW/CCW`
- 驱动器根据脉冲数和频率执行位置或速度控制
- 成本低，结构直观，适合 2 到 8 轴中小型设备

适合：

- 传统步进系统
- 支持脉冲指令输入的伺服系统
- 对总线同步要求不极端的设备

注意点：

- 同步能力通常弱于 EtherCAT 架构
- 接线较多
- 高级诊断能力通常不如总线型系统

### 3.2 模拟量或混合型运动控制卡

特点：

- 输出 `+/-10V` 模拟指令，或同时支持脉冲与模拟量
- 更适合一些高性能闭环控制或历史项目升级

适合：

- 精度要求更高的闭环伺服系统
- 老设备改造

注意点：

- 调试门槛高于纯脉冲系统
- 需要理解闭环参数、跟随误差和伺服环

### 3.3 EtherCAT 主站卡或软运动平台

特点：

- 通过 EtherCAT 管理驱动器、远程 I/O、编码器和扩展模块
- 多轴同步能力强，扩展性强，诊断能力好
- 现代设备中越来越常见

适合：

- 8 轴以上、多工站、多模组同步设备
- 需要视觉飞拍、电子凸轮、电子齿轮、严格节拍控制的设备
- 希望统一管理驱动器参数和状态的系统

注意点：

- 需要理解 `CoE`、`PDO`、`SDO`、`CiA 402`、`Distributed Clocks`
- 现场网线、拓扑和供电质量会直接影响系统稳定性

### 3.4 网络型运动控制卡

特点：

- 卡本体可以通过以太网连接 PC
- 轴控与 I/O 扩展常通过 EtherCAT、CAN 或专用协议扩展

适合：

- 希望减少 PCI/PCIe 插卡依赖
- 需要更灵活布线的设备

注意点：

- 这类产品名称里虽然也叫“运动控制卡”，但在工程使用上更接近“网络型控制器”

## 4. 你需要先搞懂的 4 条链路

### 4.1 信号链

`上位机命令 -> 运动控制卡/主站 -> 驱动器 -> 电机 -> 机械机构`

### 4.2 反馈链

`编码器/光栅尺 -> 驱动器或控制卡 -> 上位机状态/实际位置`

### 4.3 安全链

`急停/门禁/安全继电器 -> STO/断使能 -> 机械安全停机`

### 4.4 诊断链

`日志 -> 报警码 -> I/O 状态 -> 总线状态 -> 轴状态 -> 故障复盘`

很多项目做不稳，不是因为不会运动，而是因为没有把这 4 条链路连成闭环。

## 5. PCBase 开发最典型的工作流

1. 安装运动控制卡驱动和厂商 SDK。
2. 确认卡被系统识别、固件版本正确、示例程序可运行。
3. 建轴，配置单位换算、限位、原点、速度、加减速。
4. 接通 Servo ON，完成报警复位和原点回归。
5. 做点动、单轴绝对定位、相对定位。
6. 再做双轴直线插补、圆弧插补、多轴同步。
7. 最后再接视觉、MES、配方和工艺逻辑。

如果一开始就把视觉、数据库、扫码、工艺流程和运动全部绑在一起，调试会非常痛苦。

## 6. 初学者应该先会什么

### 必备知识

- 电机、驱动器、编码器三者关系
- 限位、原点、急停、使能、报警复位
- 单轴绝对移动和相对移动
- 速度、加速度、减速度、S 曲线
- 脉冲当量、编码器分辨率、机械导程

### 进阶知识

- 多轴插补
- 跟随误差
- 电子齿轮和电子凸轮
- 比较输出和位置锁存
- EtherCAT PDO/SDO 和 CiA 402 驱动状态机

## 7. 运动控制与 PLC 的区别

不是所有机器都必须上复杂运动控制平台。可以这样粗略理解：

| 场景 | 更适合 |
| --- | --- |
| 逻辑控制为主，运动少，节拍不高 | PLC |
| 多轴插补、轨迹、同步、飞拍、视觉联动 | PC-based 运动控制 |
| 高速、高精度、多模块联动、数据量大 | PC-based 运动控制更常见 |

实际工程里，两者也常混合使用，例如：

- PLC 做整机逻辑与安全联锁
- PC 做视觉、工艺和运动规划
- EtherCAT 统一挂载驱动和 I/O

## 8. 第一套学习实验建议

如果你要带新人入门，建议先做一套最小实验平台：

- 1 套工业 PC 或普通开发 PC
- 1 张运动控制卡或 1 套 EtherCAT 入门平台
- 1 台伺服驱动器
- 1 台伺服电机
- 1 个原点开关
- 2 个限位开关
- 1 个急停按钮

实验目标只做 5 件事：

1. 连接设备
2. 使能驱动器
3. 回零
4. 点动
5. 绝对定位

只要这 5 件事稳定了，你再扩到双轴、插补和视觉，成功率会高很多。

## 9. 记住这几个工程结论

- 运动控制项目，先保命，再保稳，最后再保快。
- 回零、限位、报警复位和急停联锁，比“能动起来”更重要。
- 统一抽象层比临时把 SDK 函数硬写进业务代码更重要。
- 现场最值钱的不是“会写代码”，而是“能快速定位问题并复盘”。
