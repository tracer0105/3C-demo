# 05 调试与排障清单

本章按“现场最常见问题”整理。  
真正上机后，最重要的能力不是背 API，而是能快速判断问题在：

- 接线
- 供电
- 驱动器参数
- 运动控制卡配置
- 总线状态
- 软件状态机
- 业务联锁

## 1. 上机前检查

每次第一次上电或改线后，都先做这张检查表：

| 项目 | 检查内容 |
| --- | --- |
| 电源 | 控制电源、驱动电源、24V I/O 电源是否正常 |
| 急停 | 急停回路是否闭合，释放后是否真实恢复 |
| STO | 驱动器 STO 是否释放，安全继电器是否导通 |
| 限位 | 正负限位接线方向是否正确，触发逻辑是否符合配置 |
| 原点 | 原点开关位置是否合理，触发电平是否正确 |
| 编码器 | 编码器反馈是否有值，方向是否正确 |
| 驱动器 | 参数、控制模式、报警状态是否确认 |
| 运动控制卡 | 驱动、固件、设备识别是否正常 |
| 总线 | EtherCAT 拓扑、从站顺序、网线和屏蔽是否正常 |
| 机械 | 运动范围内是否有干涉、卡滞、未锁紧部件 |

## 2. 最小调试顺序

不要一上来就点自动运行。推荐按下面顺序做：

1. 只读状态，不下运动命令
2. 报警复位
3. 使能
4. 点动
5. 停止
6. 回零
7. 绝对定位
8. 相对定位
9. 双轴联动
10. 自动流程

## 3. 常见故障速查表

### 故障 1：软件显示在线，但轴完全不动

可能原因：

- 伺服未使能
- STO 未释放
- 驱动器还在报警
- 运动模式不对
- 目标位置已下发，但被联锁拦住

优先检查：

1. `Servo ON` 是否真的到驱动器
2. 驱动器面板是否有报警码
3. 限位、急停、门禁、气压等前置条件
4. EtherCAT 驱动器的 `Controlword/Statusword`

### 故障 2：能使能，但一运动立刻报警

可能原因：

- 电机线序或编码器方向不对
- 增益过高
- 加速度过大
- 软限位或硬限位设置错误
- 机械卡死

优先检查：

- 手动低速点动是否正常
- 实际位置变化方向是否与命令一致
- 跟随误差是否快速放大

### 故障 3：回零方向反了，或一直找不到原点

可能原因：

- 原点方向参数设置错误
- 原点开关接线逻辑取反
- 先碰到限位了
- 回零模式与机构实际不匹配

优先检查：

- 原点输入在 I/O 监控里是否会变化
- 回零顺序是“找 ORG”还是“找 ORG + Z 相”
- 回零速度是否过快导致冲过开关

### 故障 4：位置会飘，重复定位不稳定

可能原因：

- 机械背隙大
- 联轴器松动
- 编码器反馈异常
- 电机抱闸释放不稳定
- 伺服增益或滤波参数不合适

优先检查：

- 机械连接是否可靠
- 电机停止后实际位置是否仍抖动
- 是否存在外力拖拽或负载变化

### 故障 5：双轴插补轨迹不圆、不直、拐角有顿挫

可能原因：

- 两轴单位换算不一致
- 加减速参数不协调
- 轨迹缓存不足
- 总线同步抖动
- 机械刚性差

优先检查：

- `PPU` 或工程单位换算
- 两轴实际速度与命令速度是否一致
- EtherCAT 周期和 jitter

### 故障 6：EtherCAT 从站在线，但上不了 OP

可能原因：

- PDO 映射不匹配
- ESI 文件或驱动器描述不一致
- 某从站供电异常
- 拓扑顺序与配置不一致

优先检查：

- 主站诊断信息
- 从站状态机停在哪个状态
- 配置下载是否成功

### 故障 7：上 OP 了，但 CiA 402 轴还是不动

可能原因：

- Operation Mode 错
- Controlword 顺序不对
- 目标位置没有更新
- 位置命令单位不对

优先检查：

- 当前模式是否为 `CSP`、`PP` 或你预期模式
- 状态字是否进入 `Operation Enabled`
- 目标位置与实际位置变化趋势

### 故障 8：比较输出或飞拍丢触发

可能原因：

- 用了软件触发而不是硬件触发
- 比较位置设置过密
- 运动速度过高
- 触发输出脉宽过窄

优先检查：

- 是否使用控制卡硬件比较功能
- 外设是否真的能接收到脉冲
- 触发时刻和编码器位置是否匹配

### 故障 9：恢复急停后，软件显示正常但轴仍不能动

可能原因：

- 安全继电器未复位
- STO 仍然有效
- 使能链没有重新打通
- 软件状态机还停留在 Fault

优先检查：

- 急停复位流程是否包含“清报警 -> 清联锁 -> 重新使能”
- 是否需要人工确认再恢复 Ready

### 故障 10：程序偶发卡顿，运动状态刷新不稳定

可能原因：

- UI 线程阻塞
- 日志同步写盘
- 视觉或网络任务抢占资源
- 总线线程和业务线程混在一起

优先检查：

- 线程模型
- CPU 占用
- 网卡驱动和电源管理
- 是否有大文件写盘、杀毒扫描、Windows 更新影响

## 4. EtherCAT 专项排障顺序

如果你的项目采用 EtherCAT，推荐按这个顺序排：

1. 物理层：网线、屏蔽、供电、接地
2. 拓扑层：从站顺序、站号、模块识别
3. 协议层：ESI、PDO、SDO、同步模式
4. 驱动层：CiA 402、模式、控制字、状态字
5. 应用层：回零、限位、业务联锁、节拍逻辑

按层排查，效率比“看到不动就乱改参数”高得多。

## 5. 现场日志最少应该看什么

每次故障至少保留以下信息：

- 故障发生时间
- 当前工站状态
- 当前轴名
- 当前运动命令
- 指令位置与实际位置
- 报警码
- 急停/限位/门禁状态
- EtherCAT 主站和从站状态

如果这些数据没有记录，很多偶发问题都会变成“只能猜”。

## 6. 调试时的 10 条经验

1. 新设备第一次上电，速度和加速度一律保守。
2. 点动方向没确认前，不要直接做长距离绝对定位。
3. 回零前先确认限位和原点输入真的会变化。
4. 所有软限位都要有明确单位，不要混用 pulse 和 mm。
5. 任何自动流程前都先给出手动模式验证路径。
6. 急停恢复流程必须写成固定 SOP。
7. 每改一次驱动器关键参数，都要留记录。
8. 机械问题和电气问题经常会互相伪装，不要只盯软件。
9. 能做硬件联锁的，不要只做软件联锁。
10. 每解决一个现场问题，都应回写到本章形成团队知识库。

## 7. 验收前检查表

在设备准备交付前，至少确认：

- 回零 50 次以上无异常
- 单轴定位与重复定位满足工艺要求
- 急停、限位、门禁、气压联锁都有效
- 断电重启后的恢复流程明确
- 日志能定位到真实故障点
- 自动节拍在目标速度下连续运行稳定

## 8. 结论

调试不是“会动了就结束”，而是：

- 能动
- 稳定
- 可恢复
- 可定位
- 可交付

真正成熟的 PCBase 项目，现场工程师和软件工程师应该共用同一份排障语言。
