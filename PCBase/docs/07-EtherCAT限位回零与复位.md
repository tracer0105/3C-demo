# 07 EtherCAT 限位、回零与复位

本章专门解决 EtherCAT 项目里最容易把人卡住的 4 件事：

- 正极限 `+EL`
- 负极限 `-EL`
- 回零 `Homing`
- 热复位 / 冷复位

如果你在现场经常遇到“轴在线但不动”“回零老失败”“限位一直亮”“复位了还是恢复不了”，这一章就是给这种情况准备的。

## 1. 本章默认前提

本文默认设备架构如下：

- 工业 PC 或控制 PC
- EtherCAT 主站卡或软主站
- 遵循 `CiA 402` 的 EtherCAT 伺服驱动器
- 远程 I/O 或驱动器自带 DI/DO
- 增量编码器或绝对值编码器伺服轴

## 2. 先把几个术语说清楚

### 正极限 `+EL`

正极限是轴朝“正方向”运动时的硬件保护边界。

触发后的基本原则是：

- 禁止继续向正方向运动
- 只允许反方向退回
- 必要时联动报警、减速停机或安全停机

### 负极限 `-EL`

负极限是轴朝“负方向”运动时的硬件保护边界。

触发后的处理逻辑与正极限对称：

- 禁止继续向负方向运动
- 只允许正方向退回

### 软正限位 / 软负限位

这是软件根据坐标值设置的运行范围。

推荐关系：

- 软限位在内
- 硬限位在外

也就是说，正常情况下先由软限位阻止错误命令；只有意外情况才应该碰到硬限位。

### 回零 `Homing`

回零的本质是“重新建立可信坐标参考”。

对增量编码器轴来说，回零几乎是必须动作。  
对绝对值编码器轴来说，虽然不一定每次开机都要回零，但仍建议保留“重新回零”功能用于维护和校准。

### 热复位 `Hot Reset`

热复位指的是：

- 不断主电源
- 不整机掉电
- 直接重置软件、主站或设备状态

它的优点是恢复快，缺点是不能清掉所有底层残留状态。

### 冷复位 `Cold Reset`

冷复位指的是：

- 控制器重启
- 驱动器重新上电
- 或整机断电后重新启动

它比热复位更彻底，但代价是更大，恢复时间更长。

## 3. 正极限怎么设置才稳

### 3.1 先统一正方向定义

不要先装限位，再回头猜哪个是正极限。  
正确顺序应是：

1. 先定义机械正方向
2. 再定义坐标正方向
3. 再确认电机实际运动正方向
4. 最后把该方向终点定义为 `+EL`

如果这一步一开始就错，后面回零、软限位、配方坐标都会跟着乱。

### 3.2 正极限安装位置建议

正极限开关的安装位置建议满足：

- 位于正常工艺行程之外
- 早于机械硬碰撞点
- 留出减速距离和释放空间

不要把正极限装得太靠里，否则正常工艺范围会被过度压缩。  
也不要装得太靠外，否则一旦误冲，来不及停车。

### 3.3 正极限输入逻辑建议

现场常见有两种接法：

- `NO` 常开
- `NC` 常闭

工程上很多团队会优先用 `NC`，原因是：

- 断线更容易暴露
- 更符合失效可检测思路

但不管用哪种，都要统一 3 处逻辑：

- 电气接线逻辑
- EtherCAT I/O 或驱动器 DI 的输入极性
- HMI 显示与软件判断逻辑

### 3.4 正极限触发后的处理建议

推荐动作：

1. 立即禁止继续向正方向发运动命令
2. 允许低速反方向 `Jog`
3. 记录触发时的命令位置和实际位置
4. 视设备风险决定是否转入报警态

不推荐的做法：

- 限位触发后仍允许自动流程继续跑
- 限位触发后靠“清状态位”继续同方向运动

## 4. 负极限怎么设置才稳

负极限的原则和正极限对称，但现场上更容易遇到这些问题：

- 回零区靠近负极限，回零一开始就撞限位
- 拖链、治具、气管在负方向更容易干涉
- 维护时手动退回空间不足

建议多检查两件事：

1. 回零路径是否会先经过负极限
2. 负方向释放限位后，是否有足够空间重新建立安全姿态

## 5. 回零怎么设置更不容易出问题

### 5.1 回零的常见实现方式

常见方式包括：

- 只找原点开关 `ORG`
- 找原点开关边沿
- `ORG + Index`
- 直接使用驱动器 `Homing Mode`
- 上位机在 `CSP/PP` 下自己完成搜索和置零

对初学者和大多数自动化设备，我更推荐：

`先找 ORG -> 再找边沿 -> 再找 Index -> 最后加 Offset`

这样做的原因是：

- 容易理解
- 复现性更好
- 零点精度通常更稳定

### 5.2 回零参数建议

至少分开配置这些参数：

- 回零方向
- 粗找速度
- 精找速度
- 回零加速度/减速度
- 原点有效电平
- 是否需要 `Index`
- `homeOffset`
- 回零超时

很多项目回零不稳，不是逻辑太复杂，而是把这些参数混成了一坨。

### 5.3 回零位置设计技巧

1. 原点开关不要贴着硬限位。
2. 回零完成零点最好离开原点开关触发区。
3. `homeOffset` 要根据机构工艺决定，不要默认写 0。
4. 回零后如果零点正好压在开关上，后续小范围点动和重复回零都会很难受。

### 5.4 回零速度技巧

建议做成两段式：

- 粗找：速度稍高，先快速接近原点区
- 精找：速度较低，用于边沿或 `Index` 精确捕获

经验上：

- 回零速度应明显低于正常生产速度
- 回零加速度也应更保守

### 5.5 回零与 EtherCAT 的关系

在 EtherCAT 项目里，回零不是单独的机械动作，而是 3 件事叠加：

1. 主站与从站状态稳定
2. 驱动器模式与 PDO 映射正确
3. 应用层正确接收“已回零”状态

所以“回零失败”可能不是原点开关坏了，而是：

- 轴没进正确模式
- 控制字状态机不完整
- 回零完成后应用层没更新 `Homed`

## 6. 正负限位与回零的设置顺序

推荐按这个顺序上机：

1. 先确认正负方向
2. 再确认正极限与负极限输入
3. 再做低速点动
4. 再做限位释放测试
5. 最后才做回零

不要在限位方向还没确认前，就直接做自动回零。

## 7. 热复位和冷复位到底怎么选

### 7.1 更适合热复位的情况

- 软件状态机卡死
- 单轴偶发错误位未清干净
- EtherCAT 主站在线，但轴状态需要重新初始化
- 改了少量参数，需要重新加载状态

热复位通常包括：

- 重新初始化运动对象
- 重新下发 `CiA 402` 模式和控制字
- 重新建立 PDO 运行态

### 7.2 更适合冷复位的情况

- 驱动器面板报警一直不消
- 从站反复上不了 `OP`
- STO、急停已经恢复，但驱动器仍不允许使能
- I/O 状态明显不一致
- 驱动器或总线设备疑似锁死

冷复位通常包括：

- 驱动器重新上电
- EtherCAT 主站重启
- 控制 PC 或整机重新启动

### 7.3 一个实用判断原则

可以用这句话判断：

`如果问题像“状态没对齐”，先热复位；如果问题像“设备没醒过来”，优先冷复位。`

## 8. EtherCAT 项目里的复位流程建议

推荐做成分层恢复 SOP：

1. 停止当前自动流程
2. 记录故障前状态
3. 清报警
4. 检查急停、STO、门禁、气压
5. 执行热复位
6. 重新使能
7. 必要时重新回零
8. 仍失败时执行冷复位
9. 冷复位后重新上电、自检、上 OP、回零

## 9. 现场最常犯的 8 个错误

1. 把正极限和负极限接反了，但软件里又用取反“硬修”。
2. 软限位设在硬限位外面，导致保护层次颠倒。
3. 原点开关离硬限位太近，回零没有退让空间。
4. 回零后把零点停在开关触发区里。
5. 只看 HMI，不看 EtherCAT I/O 实际输入位。
6. 把报警复位当成热复位，把热复位当成冷复位。
7. 冷复位后忘了重新回零。
8. 回零失败时第一反应就是提速，而不是先看模式和信号。

## 10. 最后给一个简化原则

如果你要把 EtherCAT 项目做稳，记住这 4 句话就够了：

- 正负方向先统一，再谈限位。
- 软限位在内，硬限位在外。
- 回零先求稳定，再求快。
- 复位先分层，再重启。
