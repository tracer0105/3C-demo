# PCBase 学习教程与开发文档

本目录将 `PCBase` 定义为“基于工业 PC 的运动控制开发资料库”。它面向两类读者：

- 第一次接触运动控制卡、伺服、电机与 EtherCAT 的初学者
- 需要把运动控制项目真正做起来的上位机、设备控制、自动化开发工程师

本文档的目标不是罗列一堆型号，而是把“型号、术语、选型、开发、调试”串成一条完整学习路径。你可以把它当成一个从 0 到 1 的 PC-based Motion Control 入门仓库。

## 建议阅读顺序

1. 先读“认识 PCBase 与运动控制”，建立整体概念。
2. 再读“术语表”，把高频名词和缩写扫清。
3. 然后读“常见运动控制卡型号与选型”，建立设备谱系和方案感。
4. 接着读“开发文档”，把上位机开发流程串起来。
5. 最后用“调试与排障清单”做联调和验收。

## PCBase 一句话理解

`PCBase = 工业 PC/工控机 + 运动控制卡或软运动平台 + 驱动器 + 电机 + 反馈 + I/O + 安全链 + 上位机软件`

和传统“只会做逻辑控制”的系统相比，PC-based 运动控制更强调以下能力：

- 多轴同步
- 轨迹规划
- 插补和跟随
- 与视觉、数据库、MES、上位机 UI 联动
- 更开放的软件架构和更强的算力

## 初学者最容易踩的坑

- 只看“轴数”，不看驱动方式、编码器接口、同步周期和 SDK。
- 只会调用 `MoveAbs`，但不理解回零、限位、使能、报警复位和单位换算。
- 把 Windows 上位机误当成硬实时控制器，不做线程隔离和异常联锁。
- 没有抽象轴、I/O、报警和配方，后续项目一改机型就要重写。
- 没有日志和状态机，现场故障无法复盘。

## 推荐学习路径

1. 先搞懂单轴的“使能 -> 回零 -> 点动 -> 定位 -> 停止 -> 报警复位”。
2. 再理解脉冲型控制和 EtherCAT 控制的差别。
3. 学会看电气图：限位、原点、急停、Servo ON、ALM、INP。
4. 学会做单位换算：脉冲、编码器计数、毫米、角度。
5. 学会写统一的轴抽象层，不把业务逻辑直接绑死在某家 SDK 上。
6. 再进入双轴插补、电子齿轮、电子凸轮、飞拍、比较输出等高级功能。

## 适用范围说明

- 文中的型号以“常见、代表性、易于学习”的 PC-based 运动控制产品为主。
- 既包含传统 PCI/PCIe 插卡式运动控制卡，也包含现代 EtherCAT 主站卡、软运动平台和网络型运动控制卡。
- 资料整理时间为 `2026-06-07`。产品是否在售、是否停产、是否支持某版本 Windows 或驱动器，请以下方参考资料中的官方页面为准。
